# План миграции с Vercel на Netlify

## Описание цели
Цель — перенести развертывание проекта с платформы Vercel на Netlify. Проект представляет собой Single Page Application (SPA), созданное с помощью Vite. Необходимо настроить конфигурацию для корректной маршрутизации (redirects) и сборки на Netlify.

## Требуется проверка пользователем
> [!IMPORTANT]
> Обратите внимание, что переменные окружения (Environment Variables) нужно будет вручную перенести в панель управления Netlify.

## Предлагаемые изменения

### Конфигурация Netlify

#### [NEW] [netlify.toml](file:///D:/HTML/__SHEDULE__/06.02.2026/Student-Attendance-Tracker/netlify.toml)
Создание файла конфигурации `netlify.toml` для:
- Указания команды сборки (`npm run build`).
- Указания директории публикации (`dist`).
- Настройки редиректов для SPA (перенаправление всех запросов на `index.html`).

```toml
[build]
  command = "npm run build"
  publish = "dist"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

### Удаление (опционально)

#### [DELETE] [vercel.json](file:///D:/HTML/__SHEDULE__/06.02.2026/Student-Attendance-Tracker/vercel.json)
Если файл существует, его можно удалить после успешной миграции.

## План верификации

### Ручная верификация
1.  **Создание и пуш**: Я создам файл `netlify.toml`.
2.  **Деплой (на стороне пользователя)**: Вам нужно будет:
    - Связать репозиторий с Netlify.
    - Убедиться, что Netlify подхватил настройки из `netlify.toml`.
    - **Важно:** Добавить переменные окружения (например, `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`) в настройках сайта на Netlify.
3.  **Проверка**: Открыть сайт и проверить навигацию (перезагрузить страницу на внутреннем роуте, чтобы проверить SPA redirect).
